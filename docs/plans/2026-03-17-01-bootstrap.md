# Plan 01 — Project Bootstrap

> **For agentic workers:** REQUIRED: Use `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Scaffold a production-ready Next.js project with TypeScript, Tailwind CSS, Vitest, Playwright, and GitHub Actions CI — zero business logic, just a solid foundation.

**Architecture:** Next.js App Router (client-heavy SPA — almost all logic runs in the browser, no server-side data). Domain logic isolated in `src/domain/` as pure functions. UI in `src/components/`.

**Tech Stack:** Next.js (latest stable), TypeScript (strict), Tailwind CSS v4, Vitest + Testing Library (unit/integration), Playwright (e2e), ESLint + Prettier, Husky + lint-staged, GitHub Actions.

## Task 6: GitHub Actions CI

**Files:** `.github/workflows/ci.yml`

- [ ] **Step 1: Create CI workflow**

  Create `.github/workflows/ci.yml`:

  ```yaml
  name: CI

  on:
    push:
      branches: [main]
    pull_request:
      branches: [main]

  jobs:
    quality:
      name: Lint & Type Check
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: 20
            cache: "npm"
        - run: npm ci
        - run: npm run format:check
        - run: npm run lint
        - run: npx tsc --noEmit

    unit-tests:
      name: Unit Tests
      runs-on: ubuntu-latest
      needs: quality
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: 20
            cache: "npm"
        - run: npm ci
        - run: npm run test:coverage

    e2e-tests:
      name: E2E Tests
      runs-on: ubuntu-latest
      needs: quality
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: 20
            cache: "npm"
        - run: npm ci
        - run: npx playwright install --with-deps chromium
        - run: npm run build
        - run: npm run test:e2e -- --project=chromium
          env:
            CI: true
  ```

  > Note: E2E in CI runs only Chromium to keep costs down. Local dev runs all 3 browsers.

## Task 7: CONTRIBUTING.md + CLAUDE.md update

**Files:** `CONTRIBUTING.md`, `CLAUDE.md`

- [ ] **Step 1: Create CONTRIBUTING.md**

  ```markdown
  # Contributing to QuotePart

  ## Stack

  - **Framework**: Next.js (App Router), TypeScript strict
  - **Styling**: Tailwind CSS
  - **Tests**: Vitest (unit/integration) + Playwright (e2e)
  - **CI**: GitHub Actions

  ## Setup

  ```bash
  git clone https://github.com/<org>/quotepart
  cd quotepart
  npm install
  npm run dev
  ```

  ## Branch & Commit conventions

  - Branch naming: `type/short-description` (e.g. `feat/equity-model-m3`)
  - Commits: [Conventional Commits](https://www.conventionalcommits.org/) — `feat/fix/chore/refactor/docs/test`
  - One feature = one branch = one PR

  ## Running tests

  ```bash
  npm test              # unit tests (watch mode)
  npm run test:coverage # unit tests with coverage report
  npm run test:e2e      # e2e tests (requires dev server or build)
  npm run test:e2e:ui   # e2e with interactive UI
  ```

  ## TDD workflow

  All domain logic (`src/domain/`) must be test-driven:
  1. Write a failing test
  2. Run it to confirm it fails
  3. Write minimal code to make it pass
  4. Refactor if needed
  5. Commit

  ## Architecture

  ```
  src/
    app/        — Next.js routes and layouts
    components/ — React components (UI only, no business logic)
    domain/     — Pure business logic (no React, fully testable)
    hooks/      — Custom React hooks
    lib/        — Shared utilities
  tests/
    unit/       — Vitest tests (mirrors src/ structure)
    e2e/        — Playwright tests
  ```

  ## Data privacy

  This app has no backend. All data stays client-side (URL params + localStorage). Never introduce server-side storage without explicit discussion.
  ```
