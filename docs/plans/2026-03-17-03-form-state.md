# Plan 03 ‚Äî Form & State Management

> **For agentic workers:** REQUIRED: Use `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the 4-tier data entry form with centralized state, progressive unlock logic, and mode selection (solo vs. shared). Form values drive all downstream calculations.

**Architecture:** Single `SimulationContext` (React Context + useReducer) holds the full `SimulationInput`. Each tier is an isolated component that reads/writes to context. Unlock logic is a pure function derived from state. No calculations happen here ‚Äî form feeds domain layer.

**Tech Stack:** Next.js App Router, React Context, TypeScript strict, Tailwind CSS, Vitest + Testing Library.

**Prerequisite:** Plans 01 + 02 complete.

## Key State Shape

```typescript
interface SimulationState {
  mode: "full" | "shared" | null; // null = not yet chosen
  activeTier: 0 | 1 | 2 | 3 | 4; // 0 = mode choice screen
  completedTiers: Set<1 | 2 | 3 | 4>; // tiers where user confirmed data
  skippedTiers: Set<2 | 3 | 4>; // tiers explicitly skipped
  input: Partial<SimulationInput>; // grows as tiers are filled
}
```

**Unlock logic (pure function):**

```
Tier 1 complete ‚Üí M1 + M2 unlocked
Tier 2 complete OR skipped ‚Üí M3 unlocked (or grayed)
Tier 3 complete OR skipped ‚Üí M4 unlocked (or grayed)
Tier 4 complete OR skipped ‚Üí M5 unlocked (or grayed)
```
