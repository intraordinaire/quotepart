# Plan 04 — Results Screen

> **For agentic workers:** REQUIRED: Use `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [x]`) syntax for tracking.

**Goal:** Build the results screen: comparison table (5 models), equity gauges, temporal projections, per-model detail view, and (in couple mode) domestic perception confrontation.

**Architecture:** Results are derived from `SimulationContext` state → passed through `calculate()` → rendered as pure display components. No local state except UI toggles (expanded detail view). Locked/grayed models for unfinished tiers. Click-to-detail panel per model.

**Tech Stack:** Next.js, React, TypeScript, Tailwind. No new dependencies.

**Prerequisite:** Plans 01 + 02 + 03 complete.

## Divergences from Spec / Plan

| Decision                           | Plan says                                     | Actual                                                                            | Reason                                                                                     |
| ---------------------------------- | --------------------------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| Results route                      | `/simulate/results/page.tsx` (separate route) | Tab panel inside `/simulate/page.tsx`                                             | Avoids full page navigation; keeps state in context without URL encoding (Plan 05 concern) |
| `src/lib/modelContent.ts`          | Mentioned as implementation detail            | Created as explicit file with `ModelContent` interface and `MODEL_CONTENT` record | Needed to keep static content out of the component and make it testable                    |
| `validators.ts` name check         | `input.p1.name.trim()`                        | `(input.p1.name ?? "").trim()`                                                    | Bug fix: `name` can be `undefined` when user hasn't typed a name yet                       |
| `m5-total-contribution.ts` sliders | Assumes `domesticSliders` always present      | Falls back to `DEFAULT_SLIDERS` (all 50%) when `domesticSliders` is undefined     | `calculate()` is called from Tier 1 onward; M5 must not crash before Tier 4                |
| Tier 1 income validation           | Not in Plan 04 scope                          | Added: "Suivant" blocked + inline errors when P1/P2 income = 0                    | Discovered during e2e — missing incomes caused runtime crashes in downstream models        |
