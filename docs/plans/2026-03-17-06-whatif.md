# Plan 06 ŌĆö "Et si..." Simulator

> **For agentic workers:** REQUIRED: Use `superpowers:executing-plans` to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the "Et si..." free-form scenario editor ŌĆö pre-filled with current simulation values, any field editable, results recalculate in real-time, split-screen before/after view with highlighted deltas.

**Architecture:** `WhatIfContext` wraps a local copy of `SimulationInput` initialized from the main `SimulationContext`. Edits are local ŌĆö they never mutate the "real" simulation. `calculate()` runs on every state change (debounced). Split screen shows frozen "before" snapshot on the left, live "after" on the right. "Copy link" encodes the modified state into a new URL.

**Tech Stack:** React, TypeScript, Tailwind. `useDeferredValue` or debounce for real-time calc.

**Prerequisite:** Plans 01 + 02 + 03 + 04 complete.

> **Note (from Plan 04):** The "Et si..." tab in `/simulate/page.tsx` currently renders a "coming soon" placeholder. Plan 06 should either (a) replace that placeholder with `<WhatIfShell />` directly inside the tab panel (consistent with how ResultsShell works), or (b) remove the tab and keep `/simulate/whatif` as a standalone route accessed only via the CTA. The existing CTA in `ResultsShell` already points to `/simulate/whatif`.
>
> **Note (from Plan 03):** M4 and M5 return `M4Result` / `M5Result` (not plain `ModelResult`). `WhatIfContext` should use `CalculationResults` from `calculate()` directly rather than trying to access `.modelResult` or `.optionB` without the type guards already in `ComparisonTable`.
